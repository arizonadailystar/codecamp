### Native: Pros * Can use more of the phone's capabilities * Richer experience * Can run faster * Do not always require internet or cell access * Rich set of APIs
### Native: Cons * Must be developed for multiple platforms * Costly * Time consuming to write * Bug fixes take longer to release * Approval process can be time consuming * Possibility of app rejection * Takes up space on a device
### Web apps: Pros * More cost effective * Can work on multiple devices * Search engines can access the page * Updates made in real time * No space required * Changes appear in near real time
### Web apps: Cons * Can’t use all the device capabilities (Accelerometer) * Requires internet connection * Loads slower * Look and feel seem neither Apple or Google * Limited by HTML, CSS and Javascript
### Hybrid apps A native app “wraps” the web app and embeds a browser in the downloaded application. Allows access to notifications and other device capabilities Allows the builder to have a presence in the app store without writing all new application.
